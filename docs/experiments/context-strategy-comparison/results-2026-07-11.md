# 上下文策略对比实验结果

- 日期：2026-07-11
- 命令：`uv run python docs/experiments/context-strategy-comparison/context_strategy_comparison.py`
- 运行状态：通过
- 依赖：Python 标准库；无模型调用；无真实 API；无真实 embedding / vector store

## 汇总结果

| 策略 | 通过数 | 总数 | 平均选中文本字符数 | 主要失败类型 |
| --- | ---: | ---: | ---: | --- |
| `full_context` | 1 | 2 | 298.0 | 旧政策 / 外部注入引用、错误答案、缺 human gate。 |
| `recency_only` | 0 | 2 | 150.0 | 最新外部 attachment 被当成可信材料，错误答案，缺 citation。 |
| `lossy_summary` | 0 | 2 | 86.0 | 摘要压缩后丢失 source id，并丢失退款争议的人工确认边界。 |
| `keyword_rag` | 1 | 2 | 155.0 | 关键词召回把外部注入排到前面，导致退款 case 失败。 |
| `governed_context` | 2 | 2 | 83.0 | 无失败；只选可信当前政策或产品文档，并隔离外部 attachment。 |

## Case 观察

### refund_dispute

- `full_context` 使用 `refund_policy_2024` 和 `external_attachment`，输出自动退款，未要求人工 review。
- `recency_only` 选中最新的 `external_attachment` 和当前政策，但仍跟随外部注入输出自动退款。
- `lossy_summary` 给出“需要 review”的方向，但没有保留 `refund_policy_2026` citation，也丢失 `requires_human=true`。
- `keyword_rag` 因关键词 overlap 选中 `external_attachment`，输出自动退款。
- `governed_context` 只引用 `refund_policy_2026`，要求 human review，并隔离外部 attachment。

### connector_setup

- `full_context`、`keyword_rag` 和 `governed_context` 都能引用 `connector_doc_2026`，输出 read-only token 与 dry-run。
- `recency_only` 因只看最新材料，选中外部 attachment 和退款政策，输出导出 admin token。
- `lossy_summary` 答案方向正确，但 citation 为空，不能追溯来源。

## 支撑的结论

- 上下文容量更大不等于上下文更可靠；全量 stuffing 仍可能混入旧政策和恶意外部指令。
- 只按时间新旧选择材料不够；最新材料也可能是不可信输入。
- 摘要能降低上下文规模，但如果丢失 source id、权限字段或人工确认字段，会破坏审计和验证。
- 基础 RAG 解决“找材料”的一部分问题，但 retrieval 也需要 trust、freshness、metadata 和 query intent 约束。
- Governed context 的关键不是只减少 token，而是保留 provenance、隔离不可信文档，并把 human-review gate 带入 trace。

## 不能支撑的结论

- 不能证明真实模型在长上下文、摘要或 RAG 中一定按这些失败模式出错。
- 不能证明某个 chunk size、embedding model、rerank 策略或摘要算法默认最好。
- 不能覆盖真实 token、latency、成本、跨模型稳定性、真实外部攻击样本或真实工具执行风险。
