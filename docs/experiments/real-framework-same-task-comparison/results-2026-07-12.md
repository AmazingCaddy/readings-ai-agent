## Adapter 结果

| adapter | 版本 | framework-owned capabilities | application-owned capabilities | case | 结果 |
| --- | --- | --- | --- | --- | --- |
| OpenAI Agents SDK | `0.18.2` | `FunctionTool` schema、`needs_approval` flag、tool argument validation、direct `ToolContext` invoke、fake-model `Runner` approval interruption、`RunState.approve()` resume | keyword retrieval、approval policy、refund side effect、trace redaction | 7 / 7 | pass |

## 关键观察

- OpenAI Agents SDK 能在无真实模型调用下把 `read_refund_policy` / `issue_refund` 包装为 `FunctionTool`，暴露 JSON schema、required 参数和 `needs_approval=true`，并通过直接 `ToolContext` invocation 执行 approved tool call；无效参数返回工具错误且不产生第二次 side effect。
- OpenAI Agents SDK `Runner` 能在 deterministic fake model 下处理 `needs_approval=true` 的 function tool：第一次 run 返回 1 个 approval interruption 且不执行 side effect；`RunState.approve()` 后 resume 会执行 1 次工具，生成 `function_call_output`，fake model 第二轮看到工具结果后返回 final answer。
- LangGraph 能把任务表达为 `retrieve_policy -> review_refund -> issue_refund` 的 state graph，并通过 conditional edge 在未审批时结束、审批后执行写工具。
- LlamaIndex 能直接表达检索/source-node metadata 部分；untrusted 文档会被检索到，但应用层 trust filter 没有把它放进 citations。
- Semantic Kernel 能通过 native plugin functions 表达 `read_refund_policy` 和 `issue_refund`，并暴露 function metadata。
- 四个 adapter 中，未审批退款均没有 side effect；已审批退款均产生 1 次 side effect。
- 四个 adapter 的 trace 均未泄露示例 `secret=framework-demo-token`。

## 支撑的结论

- 同一任务横向比较时，应拆开记录“框架原生表面”和“应用层补齐能力”。
- OpenAI Agents SDK、LangGraph、LlamaIndex 和 Semantic Kernel 在这个本地任务中分别暴露出不同主轴：function tool schema / approval metadata / managed runner loop、状态编排、检索/source-node metadata、插件/function metadata。
- 框架文档定位需要配合最小同任务 run 才能看清实际代码边界；但最小 run 仍只能支撑窄口径 runtime 观察。

## 不能支撑的结论

- 不能证明任一框架默认最好、最安全、最低成本或最适合初学者。
- 不能证明真实模型能正确选择工具、修正参数、抵抗 prompt injection 或稳定产生带 citation 的答案；OpenAI Agents SDK `Runner` case 使用 deterministic fake model，不代表真实 OpenAI model 行为。
- 不能证明 hosted tracing、真实 HITL UI、部署式恢复、真实并发、真实副作用事务、OpenAPI/MCP plugin、Agent/Process Framework、成本、延迟或生产可靠性。
