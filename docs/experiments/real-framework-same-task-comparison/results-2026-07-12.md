## Adapter 结果

| adapter | 版本 | framework-owned capabilities | application-owned capabilities | case | 结果 |
| --- | --- | --- | --- | --- | --- |
| LangGraph | `1.2.9` | `StateGraph`、conditional routing、compiled graph invoke | keyword retrieval、approval policy、refund side effect、trace redaction | 3 / 3 | pass |
| LlamaIndex Core | `0.14.23` | `VectorStoreIndex`、retriever、source-node metadata | trust filter、approval policy、refund side effect、trace redaction | 3 / 3 | pass |
| Semantic Kernel | `1.36.0` | plugin catalog、kernel function metadata、`Kernel.invoke()` | keyword retrieval、approval policy、refund side effect、trace redaction | 4 / 4 | pass |

## 关键观察

- LangGraph 能把任务表达为 `retrieve_policy -> review_refund -> issue_refund` 的 state graph，并通过 conditional edge 在未审批时结束、审批后执行写工具。
- LlamaIndex 能直接表达检索/source-node metadata 部分；untrusted 文档会被检索到，但应用层 trust filter 没有把它放进 citations。
- Semantic Kernel 能通过 native plugin functions 表达 `read_refund_policy` 和 `issue_refund`，并暴露 function metadata。
- 三个 adapter 中，未审批退款均没有 side effect；已审批退款均产生 1 次 side effect。
- 三个 adapter 的 trace 均未泄露示例 `secret=framework-demo-token`。

## 支撑的结论

- 同一任务横向比较时，应拆开记录“框架原生表面”和“应用层补齐能力”。
- LangGraph、LlamaIndex 和 Semantic Kernel 在这个本地任务中分别暴露出不同主轴：状态编排、检索/source-node metadata、插件/function metadata。
- 框架文档定位需要配合最小同任务 run 才能看清实际代码边界；但最小 run 仍只能支撑窄口径 runtime 观察。

## 不能支撑的结论

- 不能证明任一框架默认最好、最安全、最低成本或最适合初学者。
- 不能证明真实模型能正确选择工具、修正参数、抵抗 prompt injection 或稳定产生带 citation 的答案。
- 不能证明 hosted tracing、真实 HITL UI、部署式恢复、真实并发、真实副作用事务、OpenAPI/MCP plugin、Agent/Process Framework、成本、延迟或生产可靠性。
