# 长期记忆写入守门与治理实验结果

- 日期：2026-07-11
- 命令：`uv run python docs/experiments/memory-governance/memory_governance.py`
- 运行状态：通过
- 依赖：Python 标准库；无模型调用；无真实 memory framework

## 汇总结果

| 策略 | active memory | contamination flags | trace 泄露假 secret | 说明 |
| --- | --- | --- | --- | --- |
| `auto_write` | `api_key`、`language_preference=English`、`project_framework=MkDocs`、`skill_level=beginner` | `sensitive_secret_persisted`、`low_confidence_profile_persisted` | true | 自动写入保留了敏感信息和低置信推断。 |
| `guarded_write` | `language_preference=English`、`project_framework=MkDocs` | 无 | false | 守门策略拒绝敏感信息、低置信推断和助手猜测，并保留偏好变化历史。 |

## 关键观察

- `auto_write` 将 `api_key=sk-example-secret` 写入长期记忆，并在 trace 中泄露原始候选值。
- `auto_write` 将 `skill_level=beginner` 这种低置信模型推断写入长期记忆，可能污染后续个性化。
- `guarded_write` 拒绝 `api_key`，理由是 sensitive data must not enter long-term memory。
- `guarded_write` 拒绝 `skill_level=beginner` 和 `project_framework=LangChain`，理由是 low-confidence inferred memory requires user confirmation。
- 用户把 `language_preference` 从 Chinese 改成 English 时，两种策略都 invalidated 旧版本。
- 用户纠正 `project_framework=MkDocs` 后，guarded strategy 只保留用户纠正后的高置信事实。

## 支撑的结论

- 长期记忆写入需要区分用户明确表达、模型推断、助手猜测和敏感信息。
- 低置信推断不应自动进入长期记忆，至少需要用户确认。
- 敏感信息不应进入长期记忆，也不应以原文进入审计 trace。
- 偏好变化和事实纠正需要版本或失效历史，而不是简单覆盖。

## 不能支撑的结论

- 不能证明长期记忆提升任务表现。
- 不能证明任何真实 memory framework 的具体行为。
- 不能覆盖长期使用中的遗忘策略、权限隔离、用户删除请求、真实隐私合规或多用户数据边界。
