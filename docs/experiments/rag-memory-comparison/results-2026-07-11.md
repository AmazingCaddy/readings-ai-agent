## 关键观察

- RAG 的输出应优先绑定外部资料 citation；没有检索证据时应拒答，而不是使用 memory 猜测。
- Short-term memory 适合当前 thread/task state，例如本轮完成到哪一步；它不应替代外部知识库或长期偏好。
- Long-term memory 适合用户确认的跨会话偏好或用户纠正后的事实；自动猜测或 invalidated memory 应被跳过。
- 对敏感、未确认或没有安全来源的问题，正确行为是拒答并记录 trace，而不是从历史或记忆中推断。

## 支撑的结论

- RAG、短期记忆和长期记忆可以共同为 Agent 提供上下文，但应该在系统中分层治理。
- RAG 关注外部知识、provenance 和 citation；short-term memory 关注当前任务连续性；long-term memory 关注跨会话状态和偏好治理。
- 最小 trace 应记录 source、citations、personalization、risks 和拒答原因。

## 不能支撑的结论

- 不能证明真实 RAG stack 的 citation correctness、faithfulness、latency 或 token cost。
- 不能证明真实 memory framework 能稳定提升多会话任务质量。
- 不能覆盖用户查看/编辑/删除记忆、权限边界、隐私合规或真实模型误用记忆的概率。
