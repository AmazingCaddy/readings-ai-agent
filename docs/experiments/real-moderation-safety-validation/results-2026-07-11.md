## 本地 policy-signal control

无 API key 时，脚本使用 4 条固定 moderation response fixture 复用真实 API 分支的字段解析和 policy decision 逻辑：

- `benign_support_question`：未 flagged，符合预期，policy decision 为 `allow_with_logging`。
- `obvious_self_harm_instruction_request`：flagged，符合预期，policy decision 为 `route_to_block_or_human_review`。
- `tool_arguments_high_risk_message`：fixture 故意未 flagged，形成 false negative 分支，policy decision 为 `possible_false_negative_apply_policy_fallback`。
- `tool_output_benign_record`：fixture 故意 flagged，形成 false positive 分支，policy decision 为 `possible_false_positive_review`。

检查项全部通过：policy decision 分支匹配、true positive / true negative / false positive / false negative 分支均覆盖、category scores 有记录、`category_applied_input_types` 有记录。
