## 关键观察

- 练习项目可以被组织成小型 eval cases，而不是只靠人工看 demo 是否“像是能跑”。
- trace 字段能把失败位置分成输出格式、工具参数、检索引用、拒答和预算闸门。
- `P1-refusal` 说明 refusal 也应该进入结构化输出 schema，否则应用层很难统一处理。
- `P8-budget-block` 说明成本和预算检查应在高成本动作前执行，而不是事后统计。

## 支撑的结论

- 第 11 章的实践项目应包含验收标准、trace、失败分类和可重复运行命令。
- 初学者可以先用无模型 smoke harness 练习系统边界，再迁移到真实 Cookbook/API recipe。
- Cookbook 示例可以作为项目方向，但不能替代本地验收、成本记录和失败样例。

## 不能支撑的结论

- 不能证明 OpenAI Cookbook notebook 的依赖、成本或成功率。
- 不能证明真实 Structured Outputs、File Search/RAG、OpenAI Evals 或 Agents SDK trace/eval 的行为。
- 不能覆盖真实模型 refusal、token/latency/cost、rate-limit retry 或真实 RAG citation correctness。
