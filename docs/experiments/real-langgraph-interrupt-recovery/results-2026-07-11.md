## 关键观察

- `interrupt()` 暂停退款审批 graph。
- `Command(resume=...)` 恢复审批结果。
- `MemorySaver` checkpointer 与唯一 `thread_id`。
- `SqliteSaver` case 使用本地 SQLite checkpoint 文件，关闭第一个 saver 后重建 saver / graph，再用同一个 `thread_id` 和 `Command(resume=...)` 恢复。
- `sqlite_subprocess_restart` 使用两个独立本地 Python 进程：prepare 进程写入暂停 checkpoint 和 interrupt payload，resume 进程重建 saver / graph 后用同一个 `thread_id` 恢复。
- `sqlite_concurrent_resume` 使用两个独立本地 Python resume 进程等待同一个 gate 文件，同时恢复同一个本地 SQLite checkpoint，并把本地假工具副作用写入共享 JSONL 日志。
- `approved_once` 执行 1 次本地假退款工具。
- `duplicate_resume` 未产生第二次工具执行；本次 LangGraph 返回已完成状态 `approved_executed`，不是脚本内显式 `duplicate_blocked` 分支。因此只能支撑“不重复执行”的窄观察，不能支撑“框架默认给出重复恢复拒绝状态”。
- `rejected_resume` 不执行工具，并返回 `approval_rejected`。
- `tampered_args` 不执行工具，并返回 `argument_hash_mismatch`。
- `sqlite_process_restart` 在重建 graph 前执行 0 次工具，重建后批准恢复执行 1 次本地假退款工具；trace / interrupt payload 未发现示例 secret marker。
- `sqlite_subprocess_restart` 在 prepare 进程执行 0 次工具，resume 进程执行 1 次本地假退款工具；trace / interrupt payload 未发现示例 secret marker。
- `sqlite_concurrent_resume` 的两个 resume 进程都返回 `approved_executed`，但共享副作用日志只有 1 条记录；这只能支撑“本次本地 SQLite 并发观察没有重复执行本地假工具”，不能支撑框架默认显式拒绝重复恢复。
- `side_effect_before_interrupt` 作为设计失败记录：如果副作用放在 `interrupt()` 之前，node restart 语义可能导致重复副作用风险。
- 所有 completed / design-failure case 都未在 trace / interrupt payload 中发现本实验的示例 secret marker。

## 结论状态

- 可支撑：在 LangGraph 1.2.9 / langchain-core 1.4.9 / Python 3.12.13 / `MemorySaver` / 本地假退款工具 / 本脚本 graph shape 下，`interrupt()` + `Command(resume=...)` 能跑通最小审批恢复路径；本地应用层参数 hash 可阻断篡改；拒绝恢复不执行工具；重复 resume 没有重复执行工具；trace / interrupt payload 未泄露示例 secret marker。在 `langgraph-checkpoint-sqlite` 3.1.0 / `SqliteSaver` / 本地 SQLite checkpoint / 同一 `thread_id` 下，同进程重建 saver / graph 或由独立本地 Python 进程 resume，都可以恢复同一个暂停审批并执行一次本地假工具。两个本地 Python resume 进程同时恢复同一个暂停审批时，本次共享副作用日志只记录 1 次工具执行。
- 部分验证：这是真实 LangGraph completed run，但 SQLite restart / concurrency case 仍只是本机 SQLite 文件和本地 Python 进程，不是部署服务重启、真实并发流量、故障注入或生产事务测试。它不能证明真实数据库/支付/邮件副作用、真实审批 UI、状态表、事务幂等、框架默认参数完整性、生产审计、成本、延迟或跨框架优劣。
- 不可支撑：不能写成 LangGraph HITL 默认安全、默认幂等、默认脱敏或默认适合生产；应用层仍需保存审批状态、参数快照、执行状态、权限校验和脱敏策略。
