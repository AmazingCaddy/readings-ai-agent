## 关键观察

- `interrupt()` 暂停退款审批 graph。
- `Command(resume=...)` 恢复审批结果。
- `MemorySaver` checkpointer 与唯一 `thread_id`。
- `approved_once` 执行 1 次本地假退款工具。
- `duplicate_resume` 未产生第二次工具执行；本次 LangGraph 返回已完成状态 `approved_executed`，不是脚本内显式 `duplicate_blocked` 分支。因此只能支撑“不重复执行”的窄观察，不能支撑“框架默认给出重复恢复拒绝状态”。
- `rejected_resume` 不执行工具，并返回 `approval_rejected`。
- `tampered_args` 不执行工具，并返回 `argument_hash_mismatch`。
- `side_effect_before_interrupt` 作为设计失败记录：如果副作用放在 `interrupt()` 之前，node restart 语义可能导致重复副作用风险。
- 5 个 case 都未在 trace / interrupt payload 中发现本实验的示例 secret marker。

## 结论状态

- 可支撑：在 LangGraph 1.2.9 / langchain-core 1.4.9 / Python 3.12.13 / `MemorySaver` / 本地假退款工具 / 本脚本 graph shape 下，`interrupt()` + `Command(resume=...)` 能跑通最小审批恢复路径；本地应用层参数 hash 可阻断篡改；拒绝恢复不执行工具；重复 resume 没有重复执行工具；trace / interrupt payload 未泄露示例 secret marker。
- 部分验证：这是真实 LangGraph completed run，但只覆盖内存型 checkpointer 和一个最小 graph。它不能证明持久化 checkpointer、跨进程恢复、真实数据库/支付/邮件副作用、真实审批 UI、并发恢复、框架默认参数完整性、生产审计、成本、延迟或跨框架优劣。
- 不可支撑：不能写成 LangGraph HITL 默认安全、默认幂等、默认脱敏或默认适合生产；应用层仍需保存审批状态、参数快照、执行状态、权限校验和脱敏策略。
