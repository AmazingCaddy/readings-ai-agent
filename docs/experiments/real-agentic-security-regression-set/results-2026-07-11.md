## 支撑的结论

- Agentic-specific 安全 regression set 不能只覆盖 prompt injection；还应覆盖 tool invocation、tool poisoning、memory poisoning、MCP / remote tool、computer-use destructive action、runaway loop 和 inter-agent message 边界。
- 只靠 detector 的模拟策略会漏掉授权、远程工具、computer-use、runaway loop 和 trust-boundary 等非文本检测问题。
- 应区分 `block`、`require_approval`、`block_memory_write`、`downgrade_to_data` 和 `stop`。把所有风险都粗暴 `block` 会掩盖审批、记忆写入、handoff 降级和停止条件的不同工程含义。
- HITL 不能替代 sandbox / runtime containment。标准库模拟中，`policy_enforced_hitl` 仍未阻断 computer-use destructive action，只有加入 sandboxed runtime 边界的策略通过全部 case。
- 结果记录需要同时保存 expected decision、actual decision、false positive、false negative、side effect、trace redaction、secret leak、approval state 和 cost estimate。

## 不能支撑的结论

- 不能证明真实模型会提出这些工具调用，也不能证明真实模型会稳定遵守这些策略。
- 不能证明 MITRE ATLAS、OWASP、任何 detector、guardrail、HITL、sandbox、runtime containment、memory hardening 或 monitoring 在真实系统中有效。
- 不能证明 OpenAI Agents SDK、LangGraph、Semantic Kernel、MCP host、browser agent 或 computer-use 工具的真实覆盖范围。
- 不能记录真实成本、延迟、人工审批负担、真实误报/漏报或跨框架差异。
