## 检查项覆盖

| rule | naive 缺口 | governed 记录 |
| --- | --- | --- |
| `action_trace` | 缺 timestamp，部分步骤缺 selector / coordinates。 | url、action type、selector / coordinates、action result、timestamp。 |
| `page_state` | 缺 DOM / screenshot / before-after state。 | DOM hash、screenshot hash、before state、after state。 |
| `approval_for_side_effects` | 缺 risk level、approval required/status，提交动作直接执行。 | 高风险提交阻断为 pending，上传在 approved 后执行。 |
| `profile_isolation` | 缺 profile、test account、cookie scope、domain allowlist。 | isolated test profile、test account、demo.local cookie scope、domain allowlist。 |
| `file_upload_control` | 缺 file name、file type allow、upload approval。 | file metadata、file type policy、upload approval。 |
| `external_content_untrusted` | 缺 external content / injection / tool-result boundary。 | 记录网页内容为数据，忽略外部 `ignore_approval_and_click` 指令。 |
| `sensitive_trace_redaction` | 缺脱敏字段，且标记 secret leaked。 | sensitive input / cookie redaction，trace 只保留 hash 或 metadata。 |
| `failure_classification` | 缺 expected/actual outcome、failure type、recovery action。 | 记录 expected/actual、failure type 和 recovery action。 |

## 支撑的结论

- 浏览器 Agent 的 trace 不能只保存最终文本；至少要能回放 URL、动作类型、selector / coordinates、动作结果和时间。
- 对网页任务做 eval 时，应记录 DOM / screenshot state 或等价快照标识，尤其是提交、上传、购物、付款、删除和发布前后。
- 高风险浏览器动作需要 approval state；人工确认应区分 not required、pending、approved、rejected，而不是一个笼统布尔值。
- 浏览器练习应优先使用隔离 profile、测试账号、限定 cookie scope 和 domain allowlist。
- 文件上传要有独立策略，至少记录文件名/类型、是否允许、是否已审批，不应把完整文件内容默认写入 trace。
- 网页文字、截图/OCR 和工具结果应作为不可信数据处理；外部指令不能直接改变系统规则或跳过审批。
- Trace 本身可能包含 cookie、表单值、文件名、截图和 secret，需要脱敏、保留期限说明和泄漏检查。

## 不能支撑的结论

- 不能证明 Browser Use、Playwright、Anthropic computer use、任意 browser agent 或任意模型的真实任务成功率。
- 不能证明真实点击精度、坐标/滚动可靠性、screenshot classifier 拦截率、CAPTCHA/2FA/stealth 处理、成本、延迟、合规或生产可靠性。
- 不能替代真实 demo site、真实 trace.zip、真实 DOM/screenshot state、真实模型动作和人工复核。
