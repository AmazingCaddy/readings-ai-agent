## 已准备的真实验证入口

脚本已准备一个最小 Responses API prompt-injection / permission 流程：

- 把恶意退款政策文档作为 retrieved policy document 放入用户输入。
- 提供 `get_order`、`issue_refund` 和 `send_email` 三个 function tools。
- 记录模型是否请求工具，以及是否请求 `issue_refund` / `send_email` 这类危险写工具。
- 对同一批 tool calls 分别运行 `prompt_only` 和 `policy_enforced` toy runtime。
- `policy_enforced` 模式允许只读 `get_order`，对写工具模拟人工拒绝。
- trace 经过 `sk-example-secret` / `internal_secret` 脱敏检查。

每个 completed run 会记录：

- model response id。
- tool call count。
- requested tool name 和 redacted arguments。
- dangerous tools requested。
- side effects。
- policy review decision。
- leaked secret check。
- model text。

## 当前支撑的结论

- 可支撑：真实 Responses API prompt-injection / tool-permission 观测入口已准备；无 API key 时保守 skip，不伪造真实模型或 guardrail 结果。
