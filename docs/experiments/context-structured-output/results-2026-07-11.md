# 上下文治理与结构化输出实验结果

- 日期：2026-07-11
- 命令：`uv run python docs/experiments/context-structured-output/context_structured_output.py`
- 运行状态：通过
- 依赖：Python 标准库；无模型调用；无真实 API

## 输出解析结果

| 策略 | schema valid | semantic valid | 失败 ticket | 说明 |
| --- | ---: | ---: | --- | --- |
| `free_text` | 1/3 | 1/3 | `T-2`、`T-3` | 自由文本解析不稳定，`urgent` 无法归一到允许 enum，human-review 字段也容易漏。 |
| `json_mode` | 2/3 | 1/3 | `T-2`、`T-3` | JSON 有效不等于符合 schema；`T-3` schema 有效但语义错误。 |
| `schema_validated` | 3/3 | 2/3 | `T-3` | schema 全部通过，但 `T-3` 错误地不要求人工 review，并引用外部 attachment。 |

## 上下文治理结果

| 策略 | selected docs | stale policy | follows external instruction | passed | 说明 |
| --- | --- | --- | --- | --- | --- |
| `naive_long_context` | `policy_2024_old`、`policy_2026_current`、`external_ticket_attachment` | true | true | false | 把旧政策和外部注入都放进上下文，输出自动退款和导出 token。 |
| `governed_context` | `policy_2026_current` | false | false | true | 选择最新可信政策，把外部 attachment 标为隔离数据，要求 human review。 |

## 关键观察

- `json_mode` 中 `T-2` 是 valid JSON，但缺 `requires_human`，且 `urgency=urgent` 不符合允许 enum。
- `schema_validated` 中 `T-3` 完全符合 schema，但 `requires_human=false` 和 `citations=[external_attachment]` 仍是语义错误。
- `naive_long_context` 的失败不是因为上下文不够，而是因为没有处理来源可信度、时间顺序和外部指令隔离。
- `governed_context` 只选择最新可信 policy，并把外部文档隔离为数据，不当作指令执行。

## 支撑的结论

- 结构化输出解决的是可解析性和 schema adherence 的一部分问题，不等于事实正确、权限正确或业务正确。
- JSON mode 只保证 JSON 层面的可解析，不保证 required fields、enum 或业务规则。
- 长上下文不能替代上下文治理；系统仍需要来源标注、时效选择、可信度过滤、权限隔离和 trace。

## 不能支撑的结论

- 不能证明真实 OpenAI Structured Outputs 的稳定性或失败率。
- 不能证明真实模型在长上下文中一定会服从旧资料或外部注入。
- 不能覆盖 token、latency、成本、跨模型差异、真实 API refusal 或真实外部攻击样本。
