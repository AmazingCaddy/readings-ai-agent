## Case 结果

| case | 结果 | 观察 |
| --- | --- | --- |
| `code_agent_executes_tool_calls_from_python_snippet` | pass | `CodeAgent` 从 fake model 生成的 Python snippet 调用 `read_policy` 和 `schema_probe`，最终答案包含 trusted policy citation 和 plain-string schema probe 输出。 |
| `tool_calling_agent_executes_structured_tool_calls` | pass | `ToolCallingAgent` 在 3 次 fake-model step 中看到 `read_policy`、`schema_probe`、`final_answer` tools，并按 structured tool calls 执行。 |
| `tool_output_schema_is_not_runtime_validation` | pass | `schema_probe` 声明 `output_schema` 为 object，但返回 plain string；runtime 没有自动做 output validation。 |
| `local_python_executor_blocks_unauthorized_import` | pass | `CodeAgent` 默认 local executor 对 `import os` 返回 `Import of os is not allowed`，final answer 未执行。 |
| `trace_redaction_excludes_untrusted_secret_marker` | pass | trace 记录 retrieved / returned chunk ids，不泄露 untrusted document 中的 `secret=smolagents-demo-token`。 |

## 支撑的结论

- smolagents 适合作为 code action 与 structured tool calling 的对照样本：`CodeAgent` 在本地执行 Python code snippets，`ToolCallingAgent` 处理 structured tool calls。
- `Tool.output_schema` 可以作为 prompt / tool metadata 说明，但本地 runtime 不应被理解为会自动验证 tool output。
- 默认 local executor 具备 import allowlist 等限制表面，但这只支持“有一层限制”，不能写成 security sandbox。
- untrusted content、tool result、secret redaction 和业务权限仍然是应用层责任。

## 不能支撑的结论

- 不能证明 code agents 默认更好、smolagents benchmark claim 真实、真实模型会稳定选择正确工具或生成安全代码。
- 不能证明 Docker / E2B / Blaxel / Modal 等 sandbox 的隔离效果。
- 不能证明 Hub/MCP tools、`trust_remote_code=True`、真实成本、延迟、trace 平台质量、生产可靠性或安全合规。
