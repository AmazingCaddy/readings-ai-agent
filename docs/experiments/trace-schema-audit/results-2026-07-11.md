# Trace Schema Audit 最小实验结果

- 日期：2026-07-11
- 命令：`uv run python docs/experiments/trace-schema-audit/trace_schema_audit.py`
- 运行状态：通过
- 依赖：Python 标准库；无模型调用；无真实工具调用；无 observability 平台

## 汇总结果

| use case | 通过 trace 数 | 总 trace 数 | 说明 |
| --- | ---: | ---: | --- |
| `debug` | 4 | 5 | debug 主要需要输入、版本、工具调用、工具结果、错误和最终输出。 |
| `audit` | 2 | 5 | audit 还需要用户哈希、策略版本、审批、副作用和脱敏状态。 |
| `regression` | 1 | 5 | regression 需要 dataset、case、expected/actual、失败分类和版本。 |
| `cost_latency` | 2 | 5 | 成本和延迟分析需要 latency、token、费用估算和工具延迟。 |
| `rag` | 1 | 5 | RAG 需要 retrieval query、retrieved chunks、source/citation 和 grounded 字段。 |
| `privacy` | 2 | 5 | privacy 需要 redaction、removed fields、retention policy 和 access scope。 |

隐私失败 trace：`privacy_leaky_trace`。

完全覆盖所有用例且没有隐私失败的 trace：`eval_rag_trace`。

## Case 观察

| trace | 通过的用途 | 关键缺口 |
| --- | --- | --- |
| `minimal_log` | 无 | 缺 timestamp、model、tool、error、approval、dataset、cost、RAG 和 privacy 字段。 |
| `debug_trace` | `debug` | 可以排查工具 timeout，但缺 audit、regression、cost、RAG 和 privacy 字段。 |
| `audit_ready_trace` | `debug`、`audit`、`cost_latency`、`privacy` | 适合审计和成本分析，但缺 dataset/case/expected/actual 和 RAG citation 字段。 |
| `eval_rag_trace` | 全部 | 同时覆盖 debug、audit、regression、cost、RAG 和 privacy。 |
| `privacy_leaky_trace` | `debug` | 能 debug，但 trace 中包含 `internal_secret=sk-example-secret` 和 `customer@example.com`，隐私失败。 |

## 支撑的结论

- Trace 不能只保存最终输入输出；最小日志无法支撑工具错误排查、审计、回归、RAG 引用、成本分析或隐私治理。
- Debug trace 不是 audit trace。能排查工具错误，不代表能解释谁批准了高风险动作、是否产生副作用、采用了哪个 policy version。
- Audit trace 也不一定是 regression trace。缺 dataset、case、expected behavior、actual behavior 和 failure category 时，很难做版本对比。
- RAG / citation eval 需要 retrieval query、retrieved chunks、source IDs、scores、citation IDs 和 grounded 字段。
- Trace 隐私是独立检查项；字段够多但未脱敏，仍可能不能进入生产审计系统。

## 不能支撑的结论

- 不能证明 LangSmith、Phoenix、OpenAI Evals 或任何平台默认记录这些字段。
- 不能证明这个字段集合是所有 Agent 系统的通用标准。
- 不能覆盖真实 LLM-as-judge、人工标注、线上漂移、访问控制、数据保留合规或真实生产成本。
