# Reflection / Retry 与错误反思实验结果

- 日期：2026-07-11
- 命令：`uv run python docs/experiments/reflection-retry/reflection_retry.py`
- 运行状态：通过
- 依赖：Python 标准库；无模型调用；无真实 Agent framework

## 汇总结果

| 策略 | 成功数 | 工具调用数 | reflections | bad reflections applied | 失败任务 | 说明 |
| --- | ---: | ---: | ---: | ---: | --- | --- |
| `no_reflection` | 1/3 | 6 | 0 | 0 | `TASK-1`、`TASK-2` | 缺证据任务停留在弱结论，没有 retry。 |
| `verified_reflection_retry` | 3/3 | 14 | 2 | 0 | 无 | verifier 找出 missing evidence，retry 补读 deploy/log/migration。 |
| `unverified_reflection_memory` | 1/3 | 11 | 2 | 2 | `TASK-1`、`TASK-2` | 未验证反思让 retry 继续跳过 deploy/log/migration，错误重复。 |

## 关键观察

- `no_reflection` 在 cache 和 billing 任务中都缺关键证据，只能输出 “evidence is incomplete”。
- `verified_reflection_retry` 将反馈限定为 required evidence check：cache 任务补读 `deploy/cache.env` 和 `logs/auth.log`；billing 任务补读 migration 文件。
- `verified_reflection_retry` 成功率更高，但工具调用数从 6 增加到 14，说明 reflection/retry 有成本。
- `unverified_reflection_memory` 记录了错误经验：Issue text and service code are enough; skip deploy, logs, and migrations to reduce cost。
- 错误经验被应用到 retry 后，cache 和 billing 任务继续缺证据并失败。

## 支撑的结论

- Reflection / Critic 只有在反馈可校验、范围明确、能被 trace 记录时，才可能提升 retry 质量。
- 未验证反思不应直接写入长期记忆或后续尝试策略。
- 比较 reflection 架构时，需要记录 success、tool calls、reflection count、bad reflection applied、missing evidence 和失败原因。

## 不能支撑的结论

- 不能证明 Reflexion 或 Reflection 在真实任务中稳定提升表现。
- 不能证明真实模型能稳定生成正确反思或可靠识别错误反思。
- 不能覆盖长期 episodic memory、真实 critic、人工评审、token/latency/cost、权限确认或工具异常。

## 后续实验
