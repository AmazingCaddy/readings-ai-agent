## 支撑的结论

- 真实 `llama-index-core` runtime 可以在本地构建最小 `Document` -> `VectorStoreIndex` -> retriever 流程。
- retriever 返回的 source nodes 可以携带 `chunk_id`、`source`、`trust` 和 `score` metadata，适合进入 RAG trace / citation 记录。
- 不可信 chunk 即使被检索到，也可以在应用层 trust filter 中排除出 citations。
- 无证据问题可以保守返回 `grounded=false`，避免编造 chunk-level citation。
- 当前 trace 只记录 metadata，不记录原始恶意文本，未发现示例 secret 泄露。

## 不能支撑的结论

- 不能证明真实 LLM synthesis 会忠实引用 source nodes。
- 不能证明 citation correctness、answer faithfulness、groundedness 或 hallucination rate。
- 不能证明真实 embedding model、vector database、reranker、OpenAI File Search、metadata filter、chunk size 或 top-k 策略优劣。
- 不能证明 LlamaIndex、File Search 或任何 RAG 框架默认生产可靠、低成本、低延迟或安全。
