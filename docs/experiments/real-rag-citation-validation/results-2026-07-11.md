## 已完成的本地 verifier control

脚本当前在无 API key 时会运行一个最小 deterministic verifier control：

- 使用标准库关键词 overlap 从固定小语料中选择 chunk。
- deterministic synthesis 生成结构化 JSON：`answer`、`grounded` 和 `citations`。
- 校验 `grounded=true` 时 citation 是否只引用检索到的 `chunk_id`。
- 校验 citation quote 是否真的出现在被引用 chunk 文本中。
- 校验无证据问题是否返回 `grounded=false`，且不编造 citation。
- 对答案做最小语义检查，记录失败原因。
- 5 个 adversarial fixture 均被拒绝：unknown chunk id、grounded answer missing citation、ungrounded answer with citation、quote not in cited chunk、unsupported question marked grounded。

## 已准备的真实验证入口

配置 `OPENAI_API_KEY` 后，同一脚本会请求真实 Responses API 生成结构化 JSON，并复用上述 citation / semantic verifier。真实结果应单独记录模型、输入、输出、token / latency、citation correctness 和失败样例。
