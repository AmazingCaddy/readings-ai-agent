# Voyager-style Toy Embodied Agent 实验结果

- 日期：2026-07-11
- 命令：`uv run python docs/experiments/voyager-style-toy-agent/voyager_style_toy_agent.py`
- 运行状态：通过
- 依赖：Python 标准库；无模型调用；无 Voyager / Minecraft 环境

## 汇总结果

| 策略 | 成功数 | steps used | verified skills | sandbox rejections | stop events | 说明 |
| --- | ---: | ---: | ---: | ---: | ---: | --- |
| `no_skill_library` | 1/3 | 5 | 0 | 0 | 0 | 能收集 wood，但不会从环境错误中学会 craft pickaxe，也无法 mine stone。 |
| `unverified_skill_library` | 1/3 | 4 | 1 | 1 | 0 | 选择了未验证 `craft_pickaxe` 和含 unsafe action 的 `mine_stone` skill；失败技能被保留，暴露 skill pollution 风险。 |
| `governed_curriculum_skill_library` | 3/3 | 9 | 3 | 1 | 0 | 根据环境反馈修正 `craft_pickaxe`，拒绝并修正 unsafe `mine_stone`，自我验证后才保存 verified skill。 |

## 关键观察

- `governed_curriculum_skill_library` 记录了 `curriculum_task_started`、`skill_selected`、`action`、`environment_feedback`、`execution_error`、`skill_revised`、`self_verification` 和 `skill_stored`。
- `craft_pickaxe` 初始技能只有 `craft_pickaxe` 动作，环境返回 `pickaxe_requires_3_planks`；治理策略把它修正为 `craft_planks -> craft_sticks -> craft_pickaxe`，并在自我验证通过后保存为 verified skill。
- `mine_stone` 初始技能包含 `unsafe_spawn_stone`，sandbox 返回 `sandbox_rejected_unsafe_action`；治理策略把它修正为 `move_cave -> mine_stone`。
- 未验证技能库策略记录同样的错误，但没有修正和验证机制，因此只完成 1/3 任务。
- 本次没有触发 step budget stop event；脚本仍把 `stop_condition` 作为 trace 事件路径保留，供后续扩展长循环或 runaway loop case。

## 支撑的结论

- Voyager-style embodied / lifelong learning Agent 需要明确的环境接口、动作集合、环境反馈、执行错误、自我验证、技能库写入策略、sandbox 和停止条件。
- Skill library 不能被写成“自动提升质量”的黑盒能力；是否保存和复用技能，至少要看验证结果、失败记录、危险动作、适用环境和 trace。
- Toy environment 可作为初学者理解 embodied agent 机制的低成本入口，比直接试跑完整 Minecraft 环境更适合第一步学习。

## 不能支撑的结论

- 不能证明 Voyager 论文中的 Minecraft 结果可复现。
- 不能证明真实模型能稳定生成、修正或复用 executable skill。
- 不能证明开放式探索、自动课程或技能库在普通业务 Agent 中默认有效。
- 不能覆盖真实 API 成本、延迟、长程探索、跨环境迁移、复杂 sandbox、真实代码执行安全或生产可靠性。
