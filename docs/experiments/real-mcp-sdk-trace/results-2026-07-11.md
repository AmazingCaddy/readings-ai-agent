# Real MCP SDK Trace 结果

- 日期：2026-07-11
- 命令：`uv run --with mcp python docs/experiments/real-mcp-sdk-trace/mcp_sdk_trace.py`
- 运行状态：`completed`
- 依赖：官方 `mcp` Python SDK；本地 FastMCP stdio server；脚本化 host policy。

## 已验证的本地行为

- SDK client 通过 `ClientSession.initialize()` 连接本地 `FastMCP` stdio server，返回 protocol version `2025-11-25` 和 server capabilities。
- `tools/list` 返回 2 个 SDK tool，包括 input/output schema。
- Host policy 批准只读 `lookup_policy_section`，SDK `tools/call` 返回 `isError=false`。
- Host policy 拒绝写工具 `write_release_note`，并记录 `rejected_write_tool_forwarded=false`，因此没有把该请求转发给 server 执行。
- `resources/list` 返回 2 个 SDK resource；普通 resource 被 review 为 `ordinary_context`，恶意 resource 被 review 为 `untrusted_prompt_injection_candidate`。
- `prompts/list` / `prompts/get` 返回 1 个 prompt 和 1 条 prompt message。
- Trace 包含 12 个事件；示例 `secret=example-token` 在 host trace 中被脱敏，`leaked_secret_in_trace=false`。

## 结论状态

- 可支撑：官方 MCP Python SDK 的本地 stdio server/client 可以完成 initialize、tools、resources 和 prompts 的最小流程；工具审批、resource review、写工具拒绝和 trace 脱敏仍是 host / application 层策略。
- 部分验证：这比手写 JSON-RPC harness 更接近真实 SDK 行为，但仍只是本地 toy server 和脚本化 host policy。
- 不可支撑：不能证明真实 host UI、OAuth / URL mode elicitation、sampling、token audience validation、roots / filesystem sandbox、remote MCP connector、MCP tunnels、真实第三方 server 或跨 host 默认安全。
