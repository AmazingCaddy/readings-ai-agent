## Case 结果

| case | 结果 | 关键观察 |
| --- | --- | --- |
| `own_namespace_recall` | pass | 使用 `("memories", "user-1")` namespace search 时，只返回 user-1 namespace 下的 memory。 |
| `broad_prefix_cross_user_visible` | pass | 使用过宽的 `("memories",)` prefix search 时，可以看到 user-1 和 user-2 的 namespace；跨用户授权必须是应用层策略。 |
| `edit_records_invalidated_history` | pass | 应用层 wrapper 可以在覆盖 active memory 前，把旧版本写入 `memory_history` 并标记 `invalidated`。 |
| `delete_removes_active_recall` | pass | `delete(("memories", "user-1"), "api_token")` 后，`get` 和 user-1 namespace search 都不再返回该 active secret item。 |

## 支撑的结论

- LangGraph `InMemoryStore` 可以作为本地长期记忆 store 原语，支持 namespace、`put`、`get`、`search` 和 `delete`。
- 通过 user-scoped namespace，应用可以把不同用户的 memory 放入不同路径。
- Store 本身允许 broad prefix search；如果应用使用过宽 prefix，可能观察到多个用户 namespace。因此跨用户隔离不能只靠 prompt，也不能假设框架自动授权。
- 编辑历史、invalidated version 和敏感 trace 脱敏可以用应用层 wrapper 实现。
- 删除 active item 后，本地 store 的 `get` / `search` 不再返回该 item。

## 不能支撑的结论

- 不能证明长期记忆会提升任务质量、个性化质量或用户体验。
- 不能证明真实模型会正确使用、忽略、更新或删除 memory。
- 不能证明 LangGraph 或任何 memory framework 默认提供完整鉴权、跨用户隔离、编辑历史、合规删除、敏感过滤或 trace 脱敏。
- 不能证明真实持久化 store、跨进程恢复、组织权限、UI、成本、延迟或生产可靠性。
