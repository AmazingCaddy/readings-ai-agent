## 支撑的结论

- Repo Issue Agent 进阶练习可以先用本地 toy repo 和固定 workflow 建立对照组。
- 一个可用 toy repo 至少应记录 issue、失败测试、读取文件、diff、复跑测试、是否修改测试、trajectory 和敏感信息脱敏。
- 固定 workflow baseline 能帮助判断后续 coding agent 是否真的带来收益，而不是只看“最后测试通过”。

## 不能支撑的结论

- 本次没有运行 mini-SWE-agent、SWE-agent、OpenAI Agents SDK、LangGraph coding agent 或真实模型。
- 本次不能证明任何 coding agent 能修复 repo issue，也不能证明 bash-only agent、confirm 模式、sandbox 或某个模型默认可靠。
- 本次没有记录 token、模型调用成本、真实 agent latency、人工确认负担、容器 sandbox 隔离效果或 trajectory browser 行为。
