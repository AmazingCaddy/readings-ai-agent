## Case 观察

| case | 观察 | 结果 |
| --- | --- | --- |
| `input_guardrail_blocks_before_model` | input guardrail tripwire 触发；fake model 调用次数为 0 | passed |
| `output_guardrail_blocks_after_model` | fake model 先返回含示例 secret 的最终输出；output guardrail tripwire 随后触发；fake model 调用次数为 1 | passed |
| `tool_input_guardrail_allows_normal_refund` | 正常退款工具调用执行 1 次，工具输出为 `refund issued for order-1: 50.00` | passed |
| `tool_input_guardrail_rejects_high_amount` | 高金额工具调用返回 `refund amount requires manual approval`，本地副作用没有执行 | passed |
| `tool_output_guardrail_redacts_after_execution` | 工具先执行并产生 1 次本地副作用；tool output guardrail 把工具输出替换为 `tool output failed redaction policy` | passed |

## 能力边界

SDK runtime 直接提供的能力：

- input guardrail tripwire exception。
- output guardrail tripwire exception。
- function tool input guardrail 的 `reject_content` 行为。
- function tool output guardrail 的 `reject_content` 行为。
- function tool 上的 `needs_approval` metadata。

应用层仍负责的能力：

- fake model response 设计。
- guardrail policy function 和阈值。
- 工具副作用实现。
- trace 发布前脱敏和敏感数据审计。

## 结论

本次结果可以支撑一个窄结论：在 OpenAI Agents SDK 0.18.2 的本地 fake-model Runner 路径中，guardrail 的位置很重要。Input guardrail 可以在模型调用前阻断；output guardrail 只能在模型产出后阻断；function-tool input guardrail 可以在本地函数工具副作用前返回拒绝内容；function-tool output guardrail 发生在函数执行之后，能替换输出但不能撤销已经发生的副作用。

本次结果不能支撑更宽结论：它不调用真实模型，不评估 detector / guardrail 准确率，不覆盖 hosted tools、MCP tools、Shell / ApplyPatch tools、真实 HITL UI、生产 trace、成本或延迟。正文不能写成“用了 Agents SDK guardrails 就安全”，只能写成“guardrails 是执行路径上的机制，仍需要应用层 policy、审批状态、权限和审计设计”。
