# References 覆盖矩阵 v1

本矩阵用于检查学习手册的广度和正确性。每个核心主题至少需要 2-3 个高可信 references，其中至少 1 个应为官方文档、论文、协议规范或源码。

状态说明：

- `候选`：已列入资料清单，但未完成链接复核。
- `链接已复核`：URL 可访问，但内容尚未精读。
- `待交叉验证`：已确认来源有效，但关键结论还需要其他来源或实验支撑。
- `可入正文`：关键结论完成交叉验证，可支撑章节正文。

阅读本矩阵时要区分两个层级：`可入正文` 通常指窄口径概念、协议或工程边界已经有 references / 标准库实验支撑；表格中的“真实表现仍部分验证”指真实模型、真实 API、真实框架、成本、延迟、质量和稳定性仍需后续实验。二者不冲突。

表格中的 reference 列优先列出已经支撑当前结论的资料；标为“待建卡”或“待试跑”的条目只是后续扩展方向，不能作为当前正文中 `可入正文` 结论的证据。

Claim Boundary Consistency Audit 会检查 coverage matrix 的主题行是否保留缺口字段，以及 `可入正文` 状态摘要是否仍带有部分验证 / 待验证 / 不证明等边界措辞。这个 audit 只防文本边界回归，不验证外部事实或真实系统表现。

Source Reference Integrity Audit 会检查 source-card index、source card 元数据、章节 References 和本地 Markdown links 的结构完整性。它能防漏索引和本地断链，但不联网验证外部 URL 或资料事实。

Source URL Availability Audit 会联网检查 source card 元数据中的主来源 URL 当前是否可访问；2026-07-12 本地运行覆盖 56 张 source cards 的 73 个主 URL，全部返回 HTTP 2xx/3xx。它只补链接可达性快照，不验证资料内容正确、章节解释正确或真实系统表现。

Source Card Evidence Quality Audit 会检查进入正文的 source cards 是否具备支撑证据、风险段落和边界措辞；2026-07-12 本地运行覆盖 56 张 source cards，其中 35 张进入正文的卡片全部通过。它只补证据结构和文字边界检查，不验证外部事实或真实系统表现。

Chapter Evidence Alignment Audit 会检查 01-11 章是否具备 `已验证结论`、`待验证问题`、claim ledger / coverage matrix 控制链接，以及章节层面的边界措辞。它只补章节证据控制入口检查，不验证章节解释正确或真实系统表现。

Claim To Chapter Landing Audit 会检查 claim ledger 中每条 `可入正文` 结论是否在它声明的每个目标章节都有具体文本落点；2026-07-12 本地运行覆盖 29 条 `可入正文` 结论和 50 个声明章节落点，全部通过。它只补台账到章节正文的文本对齐检查，不验证外部事实、章节解释正确或真实系统表现。

Source To Claim Support Audit 会检查 01-11 内容章节直接引用的 source cards 是否也出现在 claim ledger `支撑资料` 列中；2026-07-12 本地运行覆盖 11 个内容章节直接引用的 55 张 source cards，全部通过。它只补章节 source card 到 claim ledger 支撑资料的文本追溯检查，不验证外部事实、章节解释正确或真实系统表现。
