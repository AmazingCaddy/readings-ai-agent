# References 覆盖矩阵 v1

本矩阵用于检查学习手册的广度和正确性。每个核心主题至少需要 2-3 个高可信 references，其中至少 1 个应为官方文档、论文、协议规范或源码。

状态说明：

- `候选`：已列入资料清单，但未完成链接复核。
- `链接已复核`：URL 可访问，但内容尚未精读。
- `待交叉验证`：已确认来源有效，但关键结论还需要其他来源或实验支撑。
- `可入正文`：关键结论完成交叉验证，可支撑章节正文。

阅读本矩阵时要区分两个层级：`可入正文` 通常指窄口径概念、协议或工程边界已经有 references / 标准库实验支撑；表格中的“真实表现仍部分验证”指真实模型、真实 API、真实框架、成本、延迟、质量和稳定性仍需后续实验。二者不冲突。

表格中的 reference 列优先列出已经支撑当前结论的资料；标为“待建卡”或“待试跑”的条目只是后续扩展方向，不能作为当前正文中 `可入正文` 结论的证据。
